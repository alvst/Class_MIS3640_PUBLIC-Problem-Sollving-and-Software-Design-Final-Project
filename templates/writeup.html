For my project, I was able to complete most of what I hoped to complete. This project had more complexity than expected. I had hoped to get access to the MusixMatch’s full API, however, after attempting to contact the company’s sales channel, I did not receive a response. Instead, I proceeded to use lyricsgenius, a python package that scrapes Genius’ lyrics.

Sentiment Analysis - The Sentiment Analysis was done using Vader Sentiment instead of NLTK analysis because I saw most other projects were using either TextBlob or NLTK Sentiment Analysis. After reading about Vader Sentiment I felt that the way it was running the analysis was more effective. I also liked that using Vader Sentiment since it differentiated the project from others (similar projects have used TextBlob or NLTK). I ran the Sentiment Analysis on the lyrics but I did not get a chance to determine how to parse the result so the results given on the website are the results that Vader Sentiment gives. In the future, I would like to add parsing to the project and create visual charts to further represent the results.

Explicitness Analysis - I also found Explicitness using profanity_check, another python package. I did this because my original Assignment 2 was able to find the Explicitness from the result returned from MusixMatch. I basically did this to provide feature parody.

Finally, a new feature I created was the ability to count the number of times a word appeared in the song. Again, this feature was inspired by other similar projects online. I, of course, also added a web interface to make it more accessible than solely the API.

Another goal of mine was to allow users to also run text analysis on song lyrics from Billboards top charts for specified years. I did not have time to implement this because the other features took far longer than I expected to implement. As a consolation, I did add the Billboard top charts so users could search those to find the songs but they would then have to implement those into the songs view so it would not be automatic. To do this, I used a module called Billboard. However, this module does not provide top charts for years out of the box. Instead, I had to find some previous Github Pull Requests that were adding this feature. Due to changes on billboards website, these changes were never implemented and no longer worked. So, to provide this functionality, I had to take what was implemented, find differences between the old and new package, and implement a new version by merging the current working version with the old version thus providing the functionality I was looking for. Again, this took several hours but I did eventually get it to work.

Again, this project had more complexity than expected which was great because it gave me additional opportunities to apply what I learned. I was able to use just about everything we learned including dictionaries, lists, API’s, and even Classes!
