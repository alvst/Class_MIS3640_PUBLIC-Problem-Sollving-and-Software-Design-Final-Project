For my final project, I would like to build on my Assignment 2 and create a web app for it while increasing the complexity. My Assignment 2 was an app that compared the sentiment of 10 different songs. It compared the top five songs from 1969 to the top songs from 2019, a 50 year gap. To increase the complexity, I would like the users to choose the years (two years would still be required). The user would also be able to select the amount of songs so would not be constrained to just 5 songs, probably with a max of 20 or so (likely limited by the API I am able to find). If the user wanted to choose and compare specific songs instead of the top songs from a period, they would also be able to select using the song name.

I would also like to look at other sentiment analysis tools to see which one is the most accurate as I am not sure how accurate NLTK sentiment analysis is for songs since they often have nuanced meanings. This may end up being only an investigation as NLTK analysis may currently be the correct or best tool.

My final deliverable would be a website and web server that offers this functionality, offering the user to choose songs or years to compare. I would like to investigate Django for the server to see if it would enable me to offer any additional functionality as well.
